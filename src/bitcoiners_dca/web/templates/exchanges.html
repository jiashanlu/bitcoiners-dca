Security notes

• All credential values are encrypted at rest with Fernet (AES-128 in CBC mode + HMAC) using DCA_SECRETS_KEY from your environment.
• Decryption happens only inside the daemon when constructing exchange clients — never in HTTP responses, logs, or YAML files.
• For OKX: prefer Read+Trade scope, IP-whitelisted to your dashboard host. Add Withdraw scope only if you enable on-chain auto-withdraw.
• Rotate any credentials that were ever pasted into a chat (Telegram, email, etc) before going live.
